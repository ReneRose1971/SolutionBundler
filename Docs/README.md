# SolutionBundler Dokumentation

Dieser `Docs`-Ordner enthält Benutzer- und Entwickl-Dokumentation für SolutionBundler. Die Dokumentation ist entlang des typischen Workflows organisiert (Solution wählen ? Scannen ? Bundle erstellen) und beinhaltet Entwickler-Informationen zu Klassen, DI und Erweiterungspunkten.

Struktur

- `README.md` - Übersicht und Navigation zu Workflow- und Entwicklerthemen
- `Workflow.md` - Schritt-für-Schritt-Workflow, welche Klassen beteiligt sind und Ausführungsreihenfolge
- `Developer.md` - Detaillierte Hinweise zu Erweiterungspunkten, Repositories, Tests und DI-Nutzung

Empfohlene Lese-Reihenfolge

1. `Workflow.md` — beschreibt den End-to-End-Ablauf und wo man starten sollte
2. `Developer.md` — tiefergehende Informationen zum Code und zu Erweiterungen

Beitragen

Füge bei Bedarf weitere Dokumente in diesen Ordner hinzu (Architektur, Tests, Release-Notes).
